# Store screenshots

Required by the Raycast Store. **Delete this README before publishing** (only `phone-*.png` files should remain).

## Specs (official Raycast guidelines)

- **Size:** 2000 × 1250 px (16:10 landscape)
- **Format:** PNG
- **Count:** max 6, recommended 3+
- **Theme:** light mode only (no dark mode)
- **Filename:** `phone-1.png`, `phone-2.png`, … `phone-6.png`

## Content rules — these will reject your PR

- **No real personal data.** Names, phone numbers, anything identifying must be fake. Screenshots are public in the Store and on GitHub.
- No other apps visible. Just Raycast.
- Same background across all screenshots.
- High-contrast background — Raycast's bundled wallpapers are recommended.

## How to capture

1. Raycast → Preferences → **Advanced** → set a **Window Capture** hotkey.
2. Run `npm run dev`.
3. Open the Phone extension in Raycast.
4. Press your Window Capture hotkey → check **"Save to Metadata"** → file lands here at the correct size.

## Suggested shots (3 is enough)

1. Full contact list, search bar focused — fake contacts only ("Mom", "Alex Doe", "Sara Lee", …).
2. Just after typing `call al` — pre-filtered list.
3. Action panel expanded showing FaceTime Audio / Video / Copy.
